Listing Guardian

Privacy Policy

Last updated: July 8, 2026

Listing Guardian ("the extension") is a local-only monitoring tool for Amazon.com product pages. This policy explains what data the extension handles and where it goes. The short version: your data stays on your device; we do not operate any server and we receive nothing.

What the extension stores (locally only)

  • Watchlist: the Amazon product pages (ASINs) you explicitly add, with your aliases, notes and per-product settings.
  • Snapshots and diffs: public fields read from those pages (price, rating, review count, rank, badges, stock wording, title, bullet points, main-image URL) and the changes computed between snapshots.
  • Alerts and settings: your alert history, subscriptions, language and schedule preferences.
  • Optional AI settings: your API endpoint, model name and API key.

All of the above is stored in your browser's local storage (IndexedDB / chrome.storage.local) on your device. It is never transmitted to the developer. There is no developer server, no analytics and no tracking of any kind. You can delete everything at any time via Settings → Delete all local data, or by removing the extension.

What the extension reads

  • Only the public Amazon.com product and review pages you explicitly added to the watchlist. The extension never accesses Seller Central, your Amazon orders, or your Amazon account data.
  • The extension checks whether an amazon.com session cookie exists to show a "connected" indicator. Cookie values are never read into storage or transmitted.
  • When you are on an Amazon product page, the extension reads that tab's URL and title so it can offer "add this page to the watchlist". It does not record your browsing history.

Optional AI features

AI summaries run only when you click an AI button, using your own API key, against the endpoint you configure (OpenAI-compatible or Anthropic-compatible). The request contains only the structured monitoring fields already visible in the panel (product facts, changes, review excerpts). It never includes cookies, API keys of other services, raw HTML, or any personal data. Your API key is stored locally and sent only to the endpoint you configured, as an authentication header.

No accounts

The extension has no sign-in and no user accounts. It never asks for, reads, or stores any personal identity information.

Data sharing

We do not sell, transfer, or share any user data with anyone. There is nothing to share — no data ever reaches us.

Changes

If this policy changes, the updated version will be published at the same URL with a new "Last updated" date.
